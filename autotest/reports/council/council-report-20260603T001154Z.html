Chairman verdict

Agree

  • The security architecture IS the deliverable. A minimal, post-deterministic runner — not a general agent. The engine has already decided PBs/ranking/colours before autonomy begins; the LLM only orchestrates + drafts captions, then queues for approval.
  • Keep the bounded, named, auditable runner framing (multi-tenancy + audit need it) but with the smallest possible tool set. Default-OFF, bounded, £0.

Structural guarantees (all test-proven)

  • Fixed allow-list registry — _dispatch(org_id, name, args) errors on any unknown name (never executed); no exec/getattr.
  • No publish, structurally — no posting credentials, no publish tool; the only write is queue_for_approval → CardStatus.QUEUE (never APPROVED/POSTED). No schedule_pending_approval — silence/timeout is NEVER approval.
  • Tenant isolation in dispatch — org_id bound once at session start, immutable, injected into every call, never read from the model; every tool re-verifies ownership.
  • Internal-id params only — no URL/path/template free strings (no SSRF / path-traversal re-entry).
  • Untrusted-content containment — ingested swimmer data + the human-authored goal inserted as inert, delimited, escaped data, never raw instructions.
  • No deterministic-engine write-back — post-rank/post-detection tools only; never pollutes the ranker / PB ledger / confidence.
  • Append-only audit log — per-org, every step recorded before a human sees anything.
  • Human-verb levels — Off / Suggest / Draft / Prepare-for-approval; the human gate fixed at publish for every level.
The one thing to do first: the structural no-publish + tenant-isolation core — _dispatch binds an immutable org_id and rejects any unknown tool; no tool can set APPROVED/POSTED or touch another org — proven by tests, before any feature tool is added.

Full advisor responses

Contrarian

Five failure surfaces: (1) prompt-injection via ingested swimmer names targets a legitimate listed tool — insert user strings as inert escaped delimited data; (2) schedule_pending_approval is a timing auto-publish bypass — approval must be positive, timeout never = approved; (3) every tool re-verifies tenant ownership; (4) URL/path/template args re-open SSRF/traversal — internal-id params only; (5) the structural guard: org_id bound once at session start, immutable, injected into dispatch, never read from the model.

First Principles

Framed wrong — a pipeline masquerading as an agent. Every real decision is already deterministic; the LLM only drafts captions. The "loop" = run the deterministic pipeline, call draft_captions(), stop. "Autonomy level" = an integer (stages before human confirmation), the gate fixed at publish. The deterministic engine is the product; the LLM is a caption writer.

Executor

Fixed REGISTRY name→callable; _dispatch errors on unknown names. 5 org-scoped tools, no path/URL args (list_pending_cards, get_card_detail, rewrite_caption, queue_for_approval never APPROVED/POSTED, get_run_summary). AutonomyLevel gates exposed tools. run_autonomy default-OFF, max_rounds-bounded, notify_pack_ready. 6 tests incl. cannot-publish, org-isolation, loop-bounded, off-by-default.

Outsider

Comfort = a "drafted 3 posts, nothing touched, click to review" notification. The gap: approval + scheduling — the tool must literally not hold posting credentials. Labels as verbs (Off / Show me ideas / Write it for me / Get it ready to post). A wrong post on a small club (real kids) is damaging — be conservative. Simplest safe v1: results → a "3 best moments and why" summary, human writes the post.

Expansionist

Flagship: the "Sunday Morning Inbox" — weekly a fully-drafted pack waiting for one-tap approve. Compounds multiplicatively with caption memory, deep-research, and the scheduler. Autonomy levels as a pricing ladder. Sport-agnostic via tool adapters. Net-new: autonomous retrospectives (approved-vs-rejected patterns → adjusted ranking weights).
